# Word Embeddings

Word embeddings are dense vector representations of words that capture **semantic meaning**, **relationships**, and **contextual similarity**. They convert human language into numerical form so neural networks can efficiently process text.

---

## Converting Words to Numbers

Machine learning models require numerical inputs. Word embeddings are a powerful way to represent words as vectors.

### Traditional Methods

**One-Hot Encoding**

* Represent each word as a vector of size *V* (vocabulary size) with a single 1 and the rest 0.
* Example: "cat" → [0, 0, 1, 0, ...]
* **Problems:** sparse, high-dimensional, no semantic meaning.

**Bag of Words (BoW)**

* Count occurrences of words in a document.
* **Problems:** ignores word order, still sparse.

### Distributed Representations (Embeddings)

Word embeddings map each word to a **dense vector** (e.g., 100–300 dimensions) that captures meaning.

Example:

* vector("king") ≈ vector("queen") - vector("woman") + vector("man")

---

## Word2Vec

Word2Vec is a method introduced by Google (2013) to learn high-quality word embeddings efficiently. It consists of two architectures:

### Continuous Bag of Words (CBOW)

* Predicts a target word from surrounding context words.

### Skip-Gram

* Predicts surrounding context words from a target word.

### Word2Vec Training Objective

For skip-gram:

$$
\max; P(\text{context words} \mid \text{target word})
$$

Softmax formulation:

$$
P(w_o \mid w_i) = \frac{e^{v_{w_o} \cdot v_{w_i}}}{\sum_{w'} e^{v_{w'} \cdot v_{w_i}}}
$$

This is computationally expensive → leads to **Negative Sampling**.

---

## Negative Sampling

A computational trick to train Word2Vec efficiently.

Instead of updating weights of all words, update:

* The true context word (positive sample)
* A small set of randomly chosen words (negative samples)

### Idea

Train a logistic regression binary classifier:

* Positive example: (target, context)
* Negative examples: (target, random words)

### Objective

$$
\log , \sigma(v_c \cdot v_t) + \sum_{v_{neg}} \log , \sigma(-v_{neg} \cdot v_t)
$$

### Benefits

* Faster training
* Works well for large vocabularies
* Produces high-quality embeddings

---

## GloVe Word Vectors

GloVe (Global Vectors) was developed by Stanford (2014).

### Key Idea

Word meaning can be derived from **global word co-occurrence statistics**.

### Co-occurrence Matrix

Let X_ij be the number of times word *j* appears in the context of word *i*.

### GloVe Objective

$$
\min \sum_{i,j} f(X_{ij}) \left( w_i^T w_j + b_i + b_j - \log X_{ij} \right)^2
$$

### Properties

* Captures linear substructures
* Uses global statistics → stable representations

---

## Sentiment Classification with Word Embeddings

### Process

1. Convert each word in a sentence to its embedding.
2. Feed the sequence into an RNN, LSTM/GRU, CNN, or Transformer.
3. Output sentiment (positive/negative).

### Why Embeddings Help

* Similar words have similar vectors
* Better generalization
* No need for manual feature engineering

---

## Debiasing Word Embeddings

Word embeddings often inherit biases from training data.

### Measuring Bias: Gender Direction

```
g = vector(he) – vector(she)
```

### Debiasing Steps

**Neutralize:** remove gender component from gender-neutral words.

**Equalize:** ensure paired words (king/queen) are equidistant from gender direction.

### Limitations

* Can't fully remove bias
* Excessive debiasing may remove useful linguistic structure

# **BERT: Bidirectional Encoder Representations from Transformers**

BERT is a **deep, bidirectional, transformer-based model** introduced by Google in 2018. It revolutionized NLP by enabling models to **understand context from both left and right directions simultaneously**.

# **Introduction**

## What is BERT?

BERT (Bidirectional Encoder Representations from Transformers) is a **language representation model** trained to understand text deeply using the **Transformer encoder** architecture.

### Key Features

* **Bidirectional attention**
* **Contextual word embeddings**
* **Pretrained on massive corpora**
* **Easily fine-tuned for many NLP tasks**

---

# **Motivation Behind BERT**

Before BERT, NLP models had limitations:

* **Static embeddings** like Word2Vec/GloVe couldn’t adapt to context.
* **LSTMs/GRUs** processed text sequentially and slowly.
* Early Transformer models were **unidirectional**, limiting context.

BERT's innovation:

* Introduces **Masked Language Modeling (MLM)** for deep bidirectional learning.
* Learns **rich, contextual representations**.
* Provides a universal architecture for many NLP tasks.

---

## **BERT Architecture**

BERT uses only the **Transformer Encoder** stack.

Each encoder block contains:

* **Multi-Head Self-Attention**
* **Add & Norm**
* **Feed-Forward Network**
* **Add & Norm**

### BERT Models

| Model      | Layers | Hidden Size | Attention Heads | Parameters |
| ---------- | ------ | ----------- | --------------- | ---------- |
| BERT Base  | 12     | 768         | 12              | 110M       |
| BERT Large | 24     | 1024        | 16              | 340M       |

---

## **Input Representation**

BERT uses three embedding types:

* **Token Embeddings**
* **Segment Embeddings** (Sentence A/B)
* **Position Embeddings** (learnable positions)

#### Special Tokens

* **[CLS]** – classification token
* **[SEP]** – separates sentences
* **[PAD]** – padding
* **[MASK]** – used in MLM training

---

## **WordPiece Tokenization**

BERT uses **subword tokenization** via WordPiece.

Examples:

```
playing → play, ##ing
unbelievable → un, ##believable
```

Benefits:

* Handles unknown words
* Reduces vocabulary size (~30k tokens)
* Produces finer-grained tokens

---

## **Pretraining Tasks**

BERT learns language understanding using two tasks.

### Masked Language Modeling (MLM)

* Randomly mask **15%** of tokens.
* Model predicts the original word.

Example:

```
The quick brown [MASK] jumps.
```

→ fox

This enables **deep bidirectional learning**.

### Next Sentence Prediction (NSP)

Used for sentence-level reasoning.

* 50%: Sentence B follows A.
* 50%: Random sentence.

Model predicts:

* "IsNext"
* "NotNext"

---

## **How BERT Learns**

**Training Pipeline**

1. Tokenize input using WordPiece.
2. Add special tokens ([CLS], [SEP]).
3. Add embeddings (token + segment + position).
4. Mask tokens (MLM).
5. Form sentence pairs (NSP).
6. Feed into Transformer encoders.
7. Compute **MLM loss** + **NSP loss**.
8. Backpropagate and update weights.

---

## **Contextual Word Embeddings**

BERT generates **different embeddings** for the same word depending on context.

Example:

* "bank" (river)
* "bank" (money)

Each produces a distinct vector.

---

## **Fine-Tuning BERT**

Fine-tuning involves:

* Adding a small task-specific layer.
* Training BERT + the new layer end-to-end.

### Common Tasks

| Task               | Method                          |
| ------------------ | ------------------------------- |
| Sentiment analysis | Use [CLS] → classifier          |
| NER                | Token-level classifier          |
| Question answering | Predict start/end positions     |
| NLI                | Classify sentence relationships |

---

## **BERT in Downstream Tasks**

### Sentiment Classification

* Use `[CLS]` embedding.
* Apply softmax.

### Named Entity Recognition (NER)

* Classify each token.

### Question Answering

Input:

```
[CLS] question [SEP] passage [SEP]
```

Output:

* Start position
* End position

---

## **Self-Attention in BERT**

Self-attention computes:

```
Attention(Q, K, V) = softmax(QKᵀ / √dk) V
```

Used to capture:

* Word relationships
* Syntax
* Long-range dependencies

## **Strengths of BERT**

* Bidirectional understanding
* Strong contextual embeddings
* Universal architecture
* Easy fine-tuning

---

## **Limitations**

* Computationally expensive
* Slow at inference
* Max sequence length = 512 tokens
* NSP is debated
