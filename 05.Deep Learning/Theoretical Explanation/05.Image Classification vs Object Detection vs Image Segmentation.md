# Image Classification vs Object Detection vs Image Segmentation

A clean, structured guide comparing three major computer vision tasks.

---

# 1. Image Classification

## Definition

Assigns a label (or multiple labels) to an entire image.
Does **not** locate where the object is.

## Output Example

```
"Dog"
```

Or multi-label:

```
["Dog", "Outdoor", "Grass"]
```

## When to Use

* You only care about presence of an object.
* No need for object position or shape.

## Applications

* Cat vs dog classification
* Medical image diagnosis
* Spam image detection
* Product category classification

## Models / Techniques

* CNNs
* VGG, ResNet, DenseNet
* MobileNet
* Vision Transformers (ViT)
* Transfer learning

---

# 2. Object Detection

## Definition

Identifies **what** objects are present and **where** they are using bounding boxes.

## Output Example

```
[person] (x=45, y=60, w=120, h=300)
[dog]    (x=200, y=80, w=150, h=220)
```

## When to Use

* You need object location.
* You need to detect multiple objects.
* Object shape precision is not required.

## Applications

* Self-driving cars
* Surveillance and security
* Robotics
* Face detection
* Traffic monitoring


## Core Concepts in Object Detection

### Bounding Boxes

Represented as:

```
(x_center, y_center, width, height)
```

### Sliding Windows

Early method—slow and inefficient.

### Convolutional Sliding Windows

Runs the CNN once over the entire image and predicts for many regions at once. Foundation of YOLO, SSD, Faster R-CNN.

### IoU (Intersection over Union)

```
IoU = area(intersection) / area(union)
```

### Non-Max Suppression (NMS)

Removes overlapping boxes with lower confidence.

### Anchor Boxes

Predefined shapes enabling detection of objects with different aspect ratios.

# Popular Object Detection Models

A detailed explanation of YOLO, SSD, and Faster R-CNN, including the core concepts that enable modern detection systems.


## YOLO (You Only Look Once)

### Overview

YOLO is a real-time object detection algorithm that treats detection as a **single regression problem**. Instead of using region proposals or sliding windows, YOLO makes predictions in one forward pass of a convolutional network.

Goal: Detect **what** objects are present and **where** they are in the image.

### Key Characteristics

* Single-stage detector
* Extremely fast (real-time performance)
* Designed for end-to-end detection
* Predicts bounding boxes and class probabilities simultaneously

### How YOLO Works

#### Grid Division

The input image is divided into an **S × S grid** (example: 13×13). Each grid cell is responsible for predicting objects whose centers fall inside the cell.

#### Predictions per Grid Cell

Each grid cell predicts:

* Bounding box coordinates (x, y, w, h)
* Objectness score
* Class probabilities

Format:

```
[p_object, b_x, b_y, b_w, b_h, class_1, class_2, ... class_k]
```

#### Anchor Boxes

YOLO uses **anchor boxes** to allow prediction of multiple bounding boxes with different shapes.

#### IoU Calculation

YOLO uses IoU to determine how well predictions match ground truth boxes.

#### Loss Function

A combined loss including:

* Localization loss
* Confidence loss
* Classification loss

#### Non-Max Suppression (NMS)

Removes redundant boxes by eliminating overlapping predictions with lower confidence.

#### Final Output

After NMS:

* Final bounding boxes
* Classes
* Confidence scores

---

## SSD (Single Shot Multibox Detector)

### Overview

SSD is a single-stage detector that predicts bounding boxes and classes from feature maps at multiple scales.

### Key Characteristics

* Very fast
* Handles small and large objects through multi-scale features

### How SSD Works

* Uses multiple CNN layers for multi-scale predictions
* Uses anchor boxes per cell
* Applies NMS for duplicate removal

### When to Use SSD

* Near real-time applications
* Tasks requiring balanced speed and accuracy

---

## Faster R-CNN

### Overview

Faster R-CNN is a two-stage detector known for its high accuracy.

### Key Characteristics

* Two-stage pipeline (RPN + classifier)
* High precision
* Slower than YOLO and SSD

### How Faster R-CNN Works

#### Feature Extraction

A CNN extracts feature maps.

#### Region Proposal Network (RPN)

Generates candidate object regions.

#### ROI Pooling

Converts proposals into fixed-size feature maps.

#### Classification and Refinement

Classifies each region and refines bounding boxes.

### When to Use Faster R-CNN

* When accuracy is the priority
* Medical imaging
* Industrial inspection

# 3. Image Segmentation

## Definition

Assigns a **label to each pixel** in the image.

Two types:

* **Semantic Segmentation**: class-level pixel labeling.
* **Instance Segmentation**: object-level pixel labeling.

## When to Use

* Object boundaries and shapes matter.
* Pixel-precise understanding required.

## Applications

* Medical imaging
* Autonomous driving
* Satellite imagery
* Agriculture

## Techniques / Models

### U-Net

Encoder–decoder structure with skip connections.

### Transpose Convolutions

Learnable upsampling used in decoders.

### Mask R-CNN

Instance segmentation on top of Faster R-CNN.

| ------------------ | --------------------- | -------------- | ----------------------- | -------------------------------- |
| **Classification** | What is in the image? | 1 label/image  | ResNet, ViT             | Simple recognition               |
| **Detection**      | What & where?         | Bounding boxes | YOLO, SSD, Faster R-CNN | Multi-object detection           |
| **Segmentation**   | What is each pixel?   | Pixel masks    | U-Net, Mask R-CNN       | Medical, driving, precise shapes |

---

# Notes

* Classification is simplest.
* Detection adds localization.
* Segmentation provides pixel-level detail.
* Detection relies on IoU, NMS, anchors.
* Segmentation uses decoders, skip connections, U-Net.

# Popular Datasets for Computer Vision

A guide comparing ImageNet, COCO, and Google Open Images: their purposes, links, use cases, and pros.

## 1. ImageNet

### Official Link

ImageNet Dataset: [https://www.image-net.org/](https://www.image-net.org/)

### Description

ImageNet is one of the most important datasets in computer vision history. It contains millions of labeled images categorized into thousands of classes.

### Scale

* 14M+ images
* 20K+ categories
* 1K classes commonly used for classification benchmarks

### Best Used For

* **Image classification training and benchmarking**
* Transfer learning for CNNs (ResNet, VGG, MobileNet)
* Pretraining models to use in downstream tasks

### Pros

* Massive dataset  excellent for generalization
* Standard benchmark for comparing models
* Clean structure for classification tasks
* Strong community + pretrained weights widely available


## 2. COCO (Common Objects in Context)

### Official Link

COCO Dataset: [https://cocodataset.org/](https://cocodataset.org/)

### Description

COCO is a dataset focused on **objects in realistic scenes**, with labels for detection, segmentation, keypoints, and more.

### Scale

* 330K+ images
* 200K labeled images
* 80 object classes
* Includes **bounding boxes, instance segmentation masks, keypoints**

### Best Used For

* **Object detection** (YOLO, Faster R-CNN)
* **Instance segmentation** (Mask R-CNN)
* **Keypoint detection** (human pose estimation)
* Real-world, multi-object environments

### Pros

* High-quality annotations
* Includes masks, not just boxes
* Diverse scenes and multiple objects per image
* Standard benchmark for detection challenges

## 3. Google Open Images

### Official Link

Open Images Dataset: [https://storage.googleapis.com/openimages/web/index.html](https://storage.googleapis.com/openimages/web/index.html)

### Description

A very large-scale dataset from Google with millions of images and rich annotations: boxes, masks, relationships, and labels.

### Scale

* ~9M images
* 600+ object classes
* Bounding boxes: 15M+
* Instance segmentation masks: large subset

### Best Used For

* Large-scale **object detection training**
* Multi-label classification
* Relationship detection
* Handling long-tail classes

### Pros

* Extremely large & diverse
* Real-world images with complex scenes
* Rich annotation types (relationships, visual attributes)
* Good for training models that need huge amounts of data
