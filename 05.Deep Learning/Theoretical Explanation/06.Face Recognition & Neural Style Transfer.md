# What Is Face Recognition?

Face recognition consists of:

### **1. Face Verification**

Checks if two images represent the same person.

### **2. Face Identification (Recognition)**

Determines *who* the person is from many possible identities.

Traditional classification fails when:

* There are millions of identities
* Few images per identity
* New identities appear over time

Thus, we use **one-shot learning**.

---

# One Shot Learning

One-shot learning means a system can recognize a person using **just one example image**.

Instead of classification, the network learns a function:

$$
f(x) → ℝ^d
$$

Where images of the same person are mapped to similar locations in embedding space.

Distance metric: typically **L2 distance**.

---

# Siamese Networks

A Siamese network consists of two identical CNNs with shared weights.

$$
x1 → CNN → f(x1)
x2 → CNN → f(x2)
$$

We compute:

$$
d = ||f(x1) - f(x2)||^2
$$

Small distance → same person.
Large distance → different person.

### Benefits

* Learns a **similarity function**
* Generalizes to unseen identities
* No retraining needed for new users

---

# Triplet Loss

Triplet loss trains embeddings using 3 images:

* Anchor (A)
* Positive (P): same identity as A
* Negative (N): different identity

Goal:
$$
||A − P||^2 + margin < ||A − N||^2
$$

### Triplet Loss Formula

$$
L = max(0, ||f(A)−f(P)||^2 − ||f(A)−f(N)||^2 + α)
$$

### Triplet Mining

Good mining chooses:

* Hard positives (look different but same identity)
* Hard negatives (look similar but different identity)

This dramatically affects training convergence.

---

# Face Verification vs Binary Classification

Binary same/different classification does **not** scale:

* Too many face pairs
* Must retrain for new users
* No generalization to unseen identities

Embedding-based verification:

* Add new users with **no retraining**
* Works with few examples
* Scales to millions

This is why systems like **FaceNet** use triplet-loss embeddings.

---

# Face Recognition Pipeline

1. **Face Detection** (MTCNN, RetinaFace, etc.)
2. **Face Alignment** (crop + normalize)
3. **Embedding Extraction** via CNN
4. **Distance Computation** with stored embeddings
5. **Thresholding** for verification or nearest neighbor for identification

# What Is Neural Style Transfer (NST)?

NST generates a new image that combines:

* **Content** of image C
* **Style** of image S

The generated image G blends structure with artistic texture.

---

# How Deep ConvNets Learn Content & Style

### Content Representation

Higher layers (e.g., conv4_2 of VGG-19):

* Capture objects, shapes, layout
* Spatial arrangement preserved

### Style Representation

Style is captured using **feature correlations** (Gram matrices):

* Brush strokes
* Colors
* Patterns
* Local textures

---

# Content Cost Function

Let:

* F = feature map of generated image
* P = feature map of content image

$$
J_content = 1/2 ||F − P||^2
$$

Uses high-level layers where structure dominates.

---

# Style Cost Function

Style uses **Gram matrices**:

$$
G_l = F_l * F_l^T
$$

Style loss for one layer:
$$
J_style_l = (1 / (4 N_l^2 M_l^2)) * || G_l(G) − G_l(S) ||^2
$$

Total style loss:
$$
J_style = Σ w_l * J_style_l
$$

---

# Total Cost Function

Final loss:

$$
J_total = α * J_content + β * J_style
$$

β is typically much larger than α to emphasize the artistic texture.

---

# Optimization in NST

NST does **not** train the CNN. Instead, the **generated image** is optimized.

Steps:

1. Initialize G (content image or noise)
2. Forward pass through pre-trained VGG
3. Compute gradients wrt G
4. Update G via gradient descent

This iteratively forms the stylized image.
