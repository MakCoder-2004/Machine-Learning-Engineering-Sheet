# Improving Deep Neural Networks

Deep neural networks (DNNs) have revolutionized modern machine learning by achieving remarkable performance in vision, speech, and natural language tasks. However, they can be difficult to train effectively due to issues like overfitting, vanishing gradients, poor convergence, and sensitivity to hyperparameters. This document explores advanced methods to **improve DNN performance**, including regularization, optimization algorithms, hyperparameter tuning, batch normalization, and multi-class classification.

---

## Regularization

Regularization helps neural networks **generalize better** by preventing overfitting — where the model memorizes training examples instead of learning the underlying patterns. Overfitting usually manifests when the model performs well on training data but poorly on unseen data.

### Why Regularization Reduces Overfitting?

Overfitting occurs when the model’s parameters become too sensitive to noise or outliers in the training data. Regularization reduces overfitting by **adding constraints or penalties** to the model’s learning process.

The goal is to minimize the cost function:

$$J(w, b) = \frac{1}{m} \sum_{i=1}^{m} L(\hat{y}^{(i)}, y^{(i)})$$

where:

* $m$: number of training examples
* $L$: loss function (e.g., cross-entropy)

Regularization adds a penalty term to discourage large weights:

$$J_{reg}(w, b) = J(w, b) + \lambda R(w)$$

Here, $R(w)$ is a regularization term and $\lambda$ is the regularization strength.

#### Why Large Weights Cause Overfitting

Large weights can make the model output highly sensitive to small input changes, leading to poor generalization. By penalizing large weights, we encourage smoother decision boundaries that generalize better.

---

### Dropout Regularization

Dropout is a **regularization technique** used to prevent **overfitting** in neural networks. It works by **randomly “dropping out” (setting to zero)** a fraction of neurons during training, which prevents the model from becoming too dependent on specific neurons.

#### Why Use Dropout?

Overfitting occurs when a neural network learns patterns that exist only in the training data but not in the unseen data. Dropout helps by:

* Making the network less sensitive to specific weights.
* Forcing the model to learn **redundant representations**.
* Acting as an ensemble method — the network behaves like it’s training multiple smaller networks.

#### How Dropout Works?

During each training iteration:

1. Each neuron has a probability `p` of being “dropped” (ignored).
2. The remaining neurons are scaled up by `1/(1-p)` to maintain the same expected output.

During **inference (testing)**, dropout is **not applied**. All neurons are active, and weights are used as-is.


#### Dropout in TensorFlow/Keras

You can easily add dropout layers in Keras using the `Dropout()` class:

```python
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, Dropout

model = Sequential([
    Dense(128, activation='relu', input_shape=(784,)),
    Dropout(0.5),  # Drops 50% of neurons during training
    Dense(64, activation='relu'),
    Dropout(0.3),  # Drops 30% of neurons
    Dense(10, activation='softmax')
])
```

#### Choosing Dropout Rate

The **dropout rate** (e.g., 0.2, 0.5) is a hyperparameter that determines how many neurons to drop.

Common recommendations:

* **0.2–0.3** for input layers
* **0.3–0.5** for hidden layers
* Avoid dropout in output layers

#### Dropout vs Other Regularization Techniques

| Technique               | Purpose               | Works By                                            |
| ----------------------- | --------------------- | --------------------------------------------------- |
| **Dropout**             | Reduce overfitting    | Randomly removes neurons during training            |
| **L2 Regularization**   | Prevent large weights | Adds penalty to loss function                       |
| **Early Stopping**      | Prevent over-training | Stops training when validation loss stops improving |
| **Batch Normalization** | Stabilize learning    | Normalizes activations per batch                    |


#### When to Use Dropout

Use dropout when:

* You notice overfitting (high training accuracy, low validation accuracy).
* You have deep or dense layers prone to overfitting.

Avoid or reduce dropout when:

* Model underfits.
* Using batch normalization (they can sometimes conflict).

### Regularization Methods

#### 1. L2 Regularization (Weight Decay)

Adds a penalty proportional to the square of the magnitude of the weights:

$$J_{reg} = J + \frac{\lambda}{2m} \sum_{l} ||W^{[l]}||_2^2$$

The gradient update rule becomes:

$$W^{[l]} = W^{[l]} - \alpha (dW^{[l]} + \frac{\lambda}{m}W^{[l]})$$

L2 regularization helps by:

* Keeping weights small and stable.
* Reducing model variance.

#### 2. L1 Regularization

Adds a penalty proportional to the absolute value of weights:

$$J_{reg} = J + \frac{\lambda}{m} \sum_{l} ||W^{[l]}||_1$$

Encourages **sparse weights** (some become zero), leading to simpler models.

#### 3. Early Stopping

Monitor validation loss during training. Stop training when validation loss starts increasing while training loss keeps decreasing.

#### 4. Data Augmentation

Expanding training data artificially (e.g., flipping, rotating, scaling images) improves robustness and generalization.

#### 5. Noise Injection

Adding small Gaussian noise to inputs or activations during training improves robustness.

## Optimization Algorithms

Optimizers are algorithms that adjust the model’s parameters to minimize the loss function. A good optimizer improves convergence speed and stability.

### Normalizing Inputs

Before training, input features are normalized:

$$x_i' = \frac{x_i - \mu_i}{\sigma_i}$$

where $\mu_i$ and $\sigma_i$ are the mean and standard deviation of the feature.

#### Why Normalize?

* Prevents one feature from dominating others.
* Speeds up gradient descent convergence.
* Helps weights learn at similar rates across all dimensions.

### Mini-Batch Gradient Descent

Rather than updating weights after every example (SGD) or after the entire dataset (batch GD), we update after a small subset — a **mini-batch**.

#### Algorithm Steps

1. Shuffle training data.
2. Split into mini-batches of size $m_{batch}$.
3. For each batch:

   * Compute gradients.
   * Update parameters.

#### Advantages

* Smooths out noisy gradient estimates.
* Speeds up training using vectorization.
* Provides regularizing effect (mini-batch noise).

### Batch Size

Batch size affects both **training speed** and **generalization**:

* Small batch → noisier updates, better generalization.
* Large batch → smoother convergence, faster on GPUs.

Common batch sizes: 32, 64, 128.

### Momentum

Momentum accelerates gradient descent by considering past gradients.

$$v = \beta v + (1 - \beta) dW$$
$$W = W - \alpha v$$

where:

* $v$: velocity (exponentially weighted average of gradients)
* $\beta$: momentum hyperparameter (typically 0.9)

#### Why Momentum Helps

Prevents oscillations and helps the model move faster along relevant directions.

### RMSprop

RMSprop adapts learning rates for each parameter based on gradient magnitudes.

$$s = \beta s + (1 - \beta)(dW)^2$$
$$W = W - \frac{\alpha dW}{\sqrt{s} + \epsilon}$$

It works well for non-stationary objectives and recurrent networks.

### Adam (Adaptive Moment Estimation)

Adam combines **Momentum** and **RMSprop**.

$$v = \beta_1 v + (1 - \beta_1)dW$$
$$s = \beta_2 s + (1 - \beta_2)(dW)^2$$

Bias correction:

$$v_{corr} = \frac{v}{1 - \beta_1^t}, \quad s_{corr} = \frac{s}{1 - \beta_2^t}$$

Parameter update:

$$W = W - \frac{\alpha v_{corr}}{\sqrt{s_{corr}} + \epsilon}$$

Typical hyperparameters:

* $\beta_1 = 0.9$
* $\beta_2 = 0.999$
* $\epsilon = 10^{-8}$

### Learning Rate Decay

Learning rate decay gradually reduces $\alpha$ over time to ensure fine-tuning near the optimum.

Common decay methods:

1. **Exponential decay:**
   $$\alpha = \alpha_0 e^{-kt}$$
2. **Step decay:** reduce $\alpha$ by a factor every few epochs.
3. **Inverse decay:**
   $$\alpha = \frac{\alpha_0}{1 + kt}$$

Helps prevent overshooting the global minimum.

### Problem of Local Optima

Deep networks have highly non-convex loss surfaces with many **local minima** and **saddle points**. At saddle points, gradients can vanish, slowing training.

### Solution of Local Optima

* Use **momentum** or **Adam** to escape flat regions.
* Initialize weights carefully (e.g., He or Xavier initialization).
* Use **batch normalization** to smooth optimization landscape.
* Use **dropout** to encourage robustness.

## Hyperparameter Tuning

Hyperparameters control the learning process. Common ones include:

### 1. Learning Rate ( $\alpha$ )

**What it is**

The **learning rate** controls how large a step the optimizer takes during each update of the model parameters.

$$
W^{[l]} := W^{[l]} - \alpha \frac{\partial J}{\partial W^{[l]}}
$$

* $\alpha$ — learning rate
* $\frac{\partial J}{\partial W^{[l]}}$ — gradient of the loss function with respect to the parameters of layer $l$

**How it affects the model**

* **Too large ($\alpha$ high):** overshoots minima, unstable or diverging training.
* **Too small ($\alpha$ low):** converges slowly, may get stuck in poor minima.
* **Optimal range:** smooth, stable convergence to a good local minimum.

**What it controls**

**Speed and stability** of training.


### 2. Regularization Strength ( $\lambda$ )

**What it is**

Regularization adds a penalty to discourage overly large weights, which reduces overfitting.

* **L2 regularization:**
  $$
  J = \frac{1}{m}\sum_{i=1}^{m} L(\hat{y}^{(i)}, y^{(i)}) + \frac{\lambda}{2m}\sum_{l}|W^{[l]}|_F^2
  $$

* **L1 regularization:**
  $$
  J = \frac{1}{m}\sum_{i=1}^{m} L(\hat{y}^{(i)}, y^{(i)}) + \frac{\lambda}{m}\sum_{l}|W^{[l]}|_1
  $$

**How it affects the model**

* **High $\lambda$:** strong regularization → smaller weights, simpler model, risk of **underfitting**.
* **Low $\lambda$:** weak regularization → model may **overfit** training data.

**What it controls**

**Model complexity** and **generalization**.


### 3. Number of Layers and Units

**What it is**

Defines the **architecture** — how deep (number of layers) and wide (number of neurons per layer) the network is.

**How it affects the model**

* **More layers:** higher capacity, captures complex patterns, risk of vanishing gradients or overfitting.
* **Fewer layers:** simpler, easier to train, may underfit.
* **More neurons:** increased representational power but greater risk of overfitting.

**What it controls**

**Model capacity** and **representation power**.

### 4. Batch Size

**What it is**

The **batch size** is the number of training examples used per parameter update.

**How it affects the model**

* **Small batch (e.g., 16–32):** noisy gradient estimates, adds regularization-like noise, often improves generalization.
* **Large batch (e.g., 256–1024):** smoother gradients, faster per-epoch convergence, but may generalize worse.

**What it controls**

Trade-off between **training speed**, **stochastic noise**, and **generalization**.


### 5. Dropout Rate

**What it is**

Dropout randomly sets a fraction of neuron outputs to zero during training to prevent co-adaptation.

If $p$ is the dropout rate, then each neuron is **kept** with probability $(1 - p)$:

$$
D^{[l]} = (A^{[l]} < 1 - p)
$$
$$
A^{[l]} = \frac{A^{[l]} \odot D^{[l]}}{1 - p}
$$

**How it affects the model**

* **High dropout (e.g., 0.7):** strong regularization, risk of underfitting.
* **Low dropout (e.g., 0.1–0.3):** mild regularization, helps reduce overfitting.

**What it controls**

**Neural co-adaptation** and **generalization**.

### Summary Table

| Hyperparameter                   | Controls          | Too Low       | Too High     | Typical Range         |
| -------------------------------- | ----------------- | ------------- | ------------ | --------------------- |
| **Learning rate ( $\alpha$ )**   | Convergence speed | Slow or stuck | Diverges     | $10^{-5}$ – $10^{-1}$ |
| **Regularization ( $\lambda$ )** | Model simplicity  | Overfit       | Underfit     | $10^{-4}$ – $10^{-1}$ |
| **Layers / Units**               | Capacity          | Underfit      | Overfit      | 3–10 layers typical   |
| **Batch size**                   | Gradient noise    | Noisy         | Sharp minima | 32–512                |
| **Dropout rate**                 | Generalization    | Overfit       | Underfit     | 0.1–0.5               |


### Strategies for Tuning

1. **Grid Search** – test combinations from a fixed grid.
2. **Random Search** – randomly sample combinations; often more efficient.
3. **Bayesian Optimization** – model performance as a function of hyperparameters and choose intelligently.

### Practical Tips

* Use logarithmic scales for ranges (e.g., $10^{-1}$ to $10^{-6}$).
* Start with coarse tuning → refine.
* Track results with tools like TensorBoard.

## Batch Normalization

Batch Normalization (BN) normalizes activations within a mini-batch.

Given input $x$:

$$\mu = \frac{1}{m} \sum_i x_i , \quad \sigma^2 = \frac{1}{m} \sum_i (x_i - \mu)^2$$

Normalize:

$$x_{norm} = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}}$$

Then scale and shift:

$$y = \gamma x_{norm} + \beta$$

Parameters $\gamma$ and $\beta$ are learnable.

### Benefits

* Reduces internal covariate shift.
* Stabilizes and accelerates training.
* Allows higher learning rates.
* Acts as regularizer (reduces need for dropout).

### Implementation Notes

BN can be applied after linear transformations and before non-linearities.

## Multi-class Classification

When there are more than two output classes, we use **softmax regression** and **cross-entropy loss**.

### Softmax Function

Softmax converts logits (raw scores) into probabilities that sum to one:

$$a_j = \frac{e^{z_j}}{\sum_{k=1}^{K} e^{z_k}}$$

where:

* $z_j$: logit for class $j$
* $K$: number of classes

### Cross-Entropy Loss

For one-hot encoded labels $y$:

$$L = - \sum_{j=1}^{K} y_j \log(a_j)$$

Minimizing this loss encourages the network to assign high probability to the correct class.

### Why Softmax?

* Produces interpretable class probabilities.
* Ensures outputs sum to 1.
* Differentiable → works well with backpropagation.

### Applications

* Image classification (e.g., MNIST, CIFAR)
* NLP (e.g., next word prediction)
* Multi-class document categorization
