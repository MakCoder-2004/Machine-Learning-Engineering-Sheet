# Neural Networks

A **Neural Network** is a supervised learning algorithm inspired by the structure and function of the human brain. It consists of layers of interconnected nodes (neurons) that process data and learn patterns. Neural networks are widely used for tasks such as classification, regression, image recognition, natural language processing, and more.

---

## The Atomic Unit: The Neuron

| Biological term  | Deep-Learning analogue | Math meaning                      |
| ---------------- | ---------------------- | --------------------------------- |
| Dendrites        | Input vector **x**     | Raw features                      |
| Synaptic weights | Weight vector **w**    | Learnable importance per feature  |
| Cell body        | Weighted sum **z**     | `z = wᵀx + b`                     |
| Nucleus          | Bias **b**             | Learnable offset (intercept)      |
| Axon hillock     | Activation **g(z)**    | Non-linear “fire or not” decision |
| Axon terminal    | Output **a**           | Signal sent to next layer         |

> **One neuron = one affine transform + one activation.**

---

## Structure of a Neural Network

A typical neural network is organized into layers:

1. **Input Layer**

   * The first layer that receives the input data.
   * Each node represents a feature of the dataset.
   * Example: For an image with 28x28 pixels, there will be 784 input nodes.

2. **Hidden Layers**

   * One or more intermediate layers that transform inputs into useful representations.
   * Each neuron applies a weighted sum followed by an **activation function**.
   * Hidden layers allow the network to capture complex, non-linear relationships.

3. **Output Layer**

   * Produces the final prediction.
   * The number of neurons corresponds to the number of classes (for classification) or 1 (for regression).
   * Example: For binary classification → 1 output neuron with a sigmoid activation.

---

## Layer = Column of Neurons

```
    ┌-----─┐
    │  a₁  │  ← neuron 1
x → │ W│a₂ │  ← neuron 2
    │  a₃  │  ← neuron 3
    └------┘
```

* **W** shape = `(#neurons, #inputs)`
* **b** shape = `(#neurons, 1)`
* **a** shape = `(#neurons, 1)`

Stack **L** such columns and you obtain a **deep network**.

---

## Types of Layers

1. **Dense (Fully Connected) Layer**

   * Every neuron is connected to every neuron in the next layer.
   * Common in feedforward neural networks.

2. **Activation Layer**

   * Applies an activation function to introduce non-linearity.
   * Common activation functions:

     * **Sigmoid**: $g(z) = \frac{1}{1+e^{-z}}$ (squashes values between 0 and 1).
     * **Tanh**: outputs between -1 and 1.
     * **ReLU**: $g(z) = \max(0, z)$ (avoids vanishing gradients).
     * **Softmax**: converts outputs into probabilities for multi-class classification.

3. **Convolutional Layer (CNNs)**

   * Applies filters to detect patterns in data (e.g., edges in images).
   * Used in image and video recognition.

4. **Pooling Layer**

   * Reduces dimensionality while preserving important features.
   * Common methods: max pooling, average pooling.

5. **Recurrent Layer (RNNs, LSTMs, GRUs)**

   * Maintains memory of previous inputs.
   * Used for sequential data (text, speech, time series).

6. **Dropout Layer**

   * Randomly drops a fraction of neurons during training to prevent overfitting.

7. **Batch Normalization Layer**

   * Normalizes inputs of each layer to stabilize training and accelerate convergence.

## How a Neural Network Works

### 1. Forward Propagation

   * Each input is multiplied by its corresponding weight and added to a bias.
   * The result is passed through an **activation function** to introduce non-linearity.
   * This process continues from layer to layer until the output is produced.

   Mathematically:
   $$ z^{[l]} = W^{[l]} a^{[l-1]} + b^{[l]} $$
   $$ a^{[l]} = g(z^{[l]}) $$

   Where:

   * $W^{[l]}$: weight matrix for layer $l$
   * $b^{[l]}$: bias vector for layer $l$
   * $g$: activation function
   * $a^{[l]}$: activation (output) of layer $l$

### 2. Loss Function

   * Measures the difference between predicted outputs and actual values.
   * Common loss functions:
     * Mean Squared Error (MSE) → regression
     * Cross-Entropy Loss → classification
        
| Task                  | Loss                                       |
| --------------------- | -------------------------------------------|
| Regression            | `MSE = 1/m Σ(ŷ-y)²`                        |
| Binary classification | `L = −1/m Σ [y log ŷ + (1−y)log(1−ŷ)]`     |
| Multi-class           | `L = −1/m Σ Σ 𝕘{y=k} log ŷ_k` (softmax-XE) |
 

### 3. Backward Propagation

   * Uses **gradient descent** to minimize the loss function.
   * Gradients of the loss with respect to weights and biases are computed using the **chain rule** of calculus.
   * Parameters are updated iteratively:

   $$ W^{[l]} = W^{[l]} - \alpha \frac{\partial J}{\partial W^{[l]}} $$
   $$ b^{[l]} = b^{[l]} - \alpha \frac{\partial J}{\partial b^{[l]}} $$

   Where $\alpha$ is the **learning rate**.

## Activation Functions

> **Definition**
> An **activation function** is a **fixed, element-wise non-linearity** that decides how “fired” a neuron becomes.
> Without it, the entire network collapses into a single linear regression.

| Function       | Equation                     | Range        | Pros                           | Cons                              | Typical use-case              |
| -------------- |------------------------------| ------------ | ------------------------------ | --------------------------------- | ----------------------------- |
| **Linear**     | `g(z)=z`                     | (-∞,∞)       | simple, differentiable         | no non-linearity                  | Output regression             |
| **Sigmoid**    | `1/(1+e⁻ᶻ)`                  | (0,1)        | probabilistic interp.          | vanishing grads, not zero-centred | Binary output, legacy         |
| **Tanh**       | `(eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ)`          | (-1,1)       | zero-centred, stronger grads   | still vanishes                    | Hidden layers in shallow nets |
| **ReLU**       | `max(0,z)`                   | [0,∞)        | fast, no vanishing, sparse     | “dying ReLU”                      | Default for hidden layers     |
| **Leaky-ReLU** | `max(αz,z)` `α=0.01`         | (-∞,∞)       | fixes dying ReLU               | extra hyper-param                 | ReLU alternative              |
| **Param-ReLU** | `max(αz,z)` `learnable α`    | same         | learnable slope                | more params                       | Very deep CNNs                |
| **ELU**        | `z if z>0 else α(eᶻ-1)`      | (-α,∞)       | smooth, zero-centred           | slower exp                        | High-accuracy CNNs            |
| **Swish**      | `z·σ(βz)`                    | ≈(-0.7,∞)    | outperforms ReLU on large sets | heavier compute                   | Large transformers            |
| **Softmax**    | `eᶻᵢ / Σⱼ eᶻⱼ`               | (0,1) sums→1 | probability simplex            | unstable if z large               | Multi-class output layer      |
| **GELU**       | `z·Φ(z)` `Φ=Gaussian CDF`    | ≈(-0.17,∞)   | smooth, SOTA NLP               | needs approximation               | BERT, GPT, ViT                |

> ### Rule-of-thumb:<br>
> Hidden → **ReLU** (or **Swish/GELU** for SOTA)<br>
> Binary output → **Sigmoid**<br>
> Multi-class output → **Softmax**<br>
> Regression output → **Linear**<br>

## Key Concepts

* **Epoch**: One complete pass through the entire dataset.
* **Batch**: A subset of the dataset used in one iteration.
* **Batch Size**: Number of samples per batch.
* **Optimizer**: Algorithm to update weights (SGD, Adam, RMSprop).
* **Overfitting**: When the model learns noise instead of the pattern → mitigated by dropout, regularization, early stopping.
* **Underfitting**: When the model is too simple to capture the data pattern.

---

## Regularisation & Training Tricks

| Trick                 | What it does                                            |
| --------------------- | ------------------------------------------------------- |
| **Dropout**           | Randomly zero neurons during training → ensemble effect |
| **Batch-Norm**        | Normalises layer inputs → stable & faster convergence   |
| **Early-stopping**    | Stop when val-loss ↑ → prevents over-fit                |
| **L2 / Weight-decay** | Penalises large weights                                 |
| **Data-augmentation** | Increases effective data (flip, crop, noise)            |

## Types of Neural Networks & When to Use Them

| Family                  | Core idea                                    | Typical shape                 | Best for                      | Key layer types            |
| ----------------------- | -------------------------------------------- | ----------------------------- | ----------------------------- | -------------------------- |
| **Feed-Forward (MLP)**  | Fully connected layers                       | `Input → Dense… → Output`     | Tabular data, simple patterns | Dense, Activation          |
| **Convolutional (CNN)** | Local filters slide over spatial grid        | `Conv → Pool → Conv… → FC`    | Images, video, spectrograms   | Conv2D, MaxPool, BatchNorm |
| **Recurrent (RNN)**     | Hidden state loops over time                 | `T time-steps, shared Wx`     | Sequences, time-series        | SimpleRNN                  |
| **LSTM / GRU**          | Gated memory cells                           | same as RNN but gated         | Long-range dependencies       | LSTM, GRU                  |
| **Encoder-Decoder**     | Compress then expand                         | `RNN→context→RNN`             | Machine translation           | LSTM + Attention           |
| **Transformer**         | Self-attention only, no recurrence           | `Multi-Head Att → FFN`        | NLP, speech, vision           | MultiHeadAtt, GELU         |
| **Auto-Encoder**        | Learn compressed representation              | `Input → bottleneck → Output` | Denoising, dimensionality     | Dense or Conv              |
| **GAN**                 | Two nets contest: Generator vs Discriminator | `G(z)→fake, D(x)→real/fake`   | Photo-real synthesis          | Conv, LeakyReLU            |
| **ResNet / DenseNet**   | Skip connections                             | `x + F(x)`                    | Very deep image nets          | ResidualBlock              |

## Practical Workflow

1. **Data** → clean, split (train/val/test), normalise
2. **Choose architecture** → CNN for images, RNN/LSTM/Transformer for text, MLP for tables
3. **Pick activations** → ReLU hidden, Softmax/Sigmoid output
4. **Pick loss** → X-entropy (class), MSE (regress)
5. **Monitor metrics** → accuracy, F1, BLEU, MAE…
6. **Train** → forward, loss, back-prop, update
7. **Tune** → learning-rate, batch-size, layers, units, dropout, epochs
8. **Evaluate** → confusion matrix, ROC, error analysis
9. **Deploy** → convert to ONNX / TFLite / TorchScript, optimise latency & size

---

## Training Process

1. Initialize weights and biases randomly.
2. Perform **forward propagation** to compute predictions.
3. Calculate the **loss function**.
4. Perform **backward propagation** to compute gradients.
5. Update parameters using **gradient descent**.
6. Repeat for multiple epochs until convergence.

## Why Use Neural Networks?

* Can model complex, non-linear relationships.
* Scalable to large datasets.
* Widely applicable (computer vision, NLP, speech recognition, etc.).

---

## When to Use Neural Networks

* When data is large and complex.
* When relationships are non-linear.
* When simpler models (like linear regression) fail to capture the complexity.
