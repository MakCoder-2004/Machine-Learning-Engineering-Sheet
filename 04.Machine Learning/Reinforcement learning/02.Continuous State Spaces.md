# Continuous State Spaces

In **Reinforcement Learning (RL)**, a **continuous state space** means that the agent’s possible states are not discrete but can take on infinitely many values — usually real numbers. This is common in environments with physical systems (e.g., robot positions, velocities, temperatures) where variables change smoothly.

---

## Definition

A **continuous state space** is defined as:

$$
S \subseteq \mathbb{R}^n
$$

where each state $s \in S$ is represented as an $n$-dimensional vector of continuous variables.

---

## Challenges

Continuous spaces make RL more complex because:

1. **Infinite possible states** → can’t store a value for each state.
2. **Function approximation** is required → we use neural networks instead of lookup tables.
3. **Exploration** is harder → sampling from a continuous range is non-trivial.

---

## Value and Policy Functions in Continuous Spaces

In continuous domains, the value function becomes a **continuous function**:

$$
V^\pi(s) = \mathbb{E}_\pi [ R_t \mid s_t = s ]
$$

and the **Q-function** is:

$$
Q^\pi(s, a) = \mathbb{E}_\pi [ R_t \mid s_t = s, a_t = a ]
$$

Both $V$ and $Q$ are typically approximated using **neural networks** (function approximators):

$$
Q_\theta(s, a) \approx Q^\pi(s, a)
$$

where $\theta$ are the parameters of the network.

---

## Function Approximation Methods

| Method                                          | Description                                                                                           |
| ----------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| **Neural Networks**                             | Deep Q-Networks (DQN) or Actor–Critic methods use NN approximators for $V$, $Q$, or policy functions. |
| **Tile Coding / Radial Basis Functions (RBFs)** | Use feature expansions to approximate continuous inputs.                                              |
| **Gaussian Processes**                          | Probabilistic approach to model uncertainty in value estimates.                                       |

---

## Common Algorithms for Continuous State Spaces

| Algorithm                                     | Type            | Description                                                              |
| --------------------------------------------- | --------------- | ------------------------------------------------------------------------ |
| **DDPG (Deep Deterministic Policy Gradient)** | Actor–Critic    | Handles continuous action and state spaces using deterministic policies. |
| **TD3 (Twin Delayed DDPG)**                   | Actor–Critic    | Improves DDPG by reducing overestimation bias.                           |
| **SAC (Soft Actor–Critic)**                   | Actor–Critic    | Maximizes entropy to encourage exploration.                              |
| **A3C / PPO**                                 | Policy Gradient | Uses policy-based methods that naturally handle continuous states.       |

## TensorFlow Implementation Example

Below is a simplified implementation using a neural network to approximate the **state-value function** for a continuous environment (e.g., `CartPole-v1` or `Pendulum-v1`).

```python
import tensorflow as tf
from tensorflow.keras import layers
import numpy as np

# Define a simple value function approximator
class ValueNetwork(tf.keras.Model):
    def __init__(self, state_dim):
        super(ValueNetwork, self).__init__()
        self.fc1 = layers.Dense(64, activation='relu')
        self.fc2 = layers.Dense(64, activation='relu')
        self.v = layers.Dense(1)

    def call(self, state):
        x = self.fc1(state)
        x = self.fc2(x)
        return self.v(x)

# Example usage
state_dim = 4  # e.g., position, velocity, angle, angular velocity
value_net = ValueNetwork(state_dim)
optimizer = tf.keras.optimizers.Adam(learning_rate=0.001)

# Training step example
@tf.function
def train_step(states, targets):
    with tf.GradientTape() as tape:
        values = value_net(states)
        loss = tf.reduce_mean(tf.square(values - targets))
    grads = tape.gradient(loss, value_net.trainable_variables)
    optimizer.apply_gradients(zip(grads, value_net.trainable_variables))
    return loss

# Dummy training loop
for _ in range(100):
    states = tf.random.normal((32, state_dim))
    targets = tf.random.uniform((32, 1))  # random rewards
    loss = train_step(states, targets)

print("Final loss:", loss.numpy())
```

This example demonstrates how continuous state inputs are passed through a neural network to estimate value functions.

---

## When to Use Continuous State Spaces

Use continuous state RL methods when:

✅ The system involves **real-world, physical parameters** (angles, forces, positions, velocities). <br>
✅ The state cannot be discretized without losing important details. <br>
✅ You want **smooth policy behavior** (e.g., robotic arm control). <br>

---

## Real-World Applications

| Domain                  | Application                                                                                 |
| ----------------------- | ------------------------------------------------------------------------------------------- |
| **Robotics**            | Controlling robotic arms, drones, or legged robots with continuous torque and joint angles. |
| **Autonomous Vehicles** | Steering, braking, and acceleration decisions based on continuous sensory input.            |
| **Finance**             | Portfolio optimization, where states and actions are continuous investment weights.         |
| **Energy Systems**      | Continuous control of grid power flow, voltage, and consumption.                            |
| **Healthcare**          | Adjusting treatment dosages or interventions based on continuous physiological measures.    |

---

## Summary

Continuous state spaces require **function approximation** for effective learning. Neural networks are the standard tool to approximate $V(s)$, $Q(s,a)$, and $\pi(a|s)$, enabling RL algorithms to handle complex, real-world environments with infinite possibilities.
