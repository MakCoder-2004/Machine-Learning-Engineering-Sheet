# State–Action Value Function

The **State–Action Value Function** (also known as the **Q-function**) is a fundamental concept in **Reinforcement Learning (RL)**.
It represents the **expected total reward** that an agent can obtain by taking a specific action in a given state and following a certain policy thereafter.

---

## Definition

The **state–action value function** under a policy $\pi$, denoted as $Q_\pi(s, a)$, is defined as:

$$
Q_\pi(s, a) = \mathbb{E}*\pi \left[ \sum*{t=0}^{\infty} \gamma^t R_{t+1} \mid S_0 = s, A_0 = a \right]
$$

Where:

| Symbol                  | Meaning                                              |
| ----------------------- | ---------------------------------------------------- |
| $s$                     | Current state                                        |
| $a$                     | Action taken at state $s$                            |
| $R_{t+1}$               | Reward received after taking action $a$              |
| $\gamma$                | Discount factor $(0 \leq \gamma \leq 1)$             |
| $\pi$                   | Policy (mapping from states to action probabilities) |
| $\mathbb{E}_\pi[\cdot]$ | Expectation when following policy $\pi$              |

---

## Bellman Equation for $Q_\pi(s, a)$

$$
Q_\pi(s, a) = \mathbb{E}*\pi \left[ R*{t+1} + \gamma Q_\pi(S_{t+1}, A_{t+1}) \mid S_t = s, A_t = a \right]
$$

Or equivalently:

$$
Q_\pi(s, a) = \sum_{s'} P(s' \mid s, a) \left[ R(s, a, s') + \gamma \sum_{a'} \pi(a' \mid s') Q_\pi(s', a') \right]
$$

---

## Bellman Optimality Equation

The **optimal state–action value function** $Q^*(s, a)$ satisfies:

$$
Q^*(s, a) = \mathbb{E} \left[ R_{t+1} + \gamma \max_{a'} Q^*(S_{t+1}, a') \mid S_t = s, A_t = a \right]
$$

or

$$
Q^*(s, a) = \sum_{s'} P(s' \mid s, a) \left[ R(s, a, s') + \gamma \max_{a'} Q^*(s', a') \right]
$$

This forms the basis of **Q-learning**, where agents iteratively approximate $Q^*$ using sampled experiences.

---

## Random (Stochastic) Environments

In **stochastic environments**, transitions and rewards are **probabilistic**:

$$
P(s', r \mid s, a)
$$

represents the joint probability of transitioning to $s'$ and receiving reward $r$ after $(s, a)$.

Then:

$$
Q_\pi(s, a) = \sum_{s'} \sum_{r} P(s', r \mid s, a) \left[ r + \gamma \sum_{a'} \pi(a' \mid s') Q_\pi(s', a') \right]
$$

This expectation accounts for randomness in the environment’s dynamics.

---

## When to Use the Q-function

Use **state–action value functions** when you need:

* **Action selection** guidance (e.g., $\epsilon$-greedy strategies)
* To evaluate or improve a policy via **Q-learning** or **SARSA**
* To handle **stochastic, sequential decision-making problems**

---

## Real-World Applications

✅ **Autonomous Driving** – learning steering/braking actions
✅ **Robotics** – fine-grained control of actuators
✅ **Finance** – adaptive trading under uncertainty
✅ **Healthcare** – treatment policy optimization
✅ **Game AI** – optimal move selection under randomness

## TensorFlow Implementation (Q-Learning Example)

Below is a minimal TensorFlow implementation demonstrating how the **state–action value function** is learned using **Deep Q-Learning**.

```python
import numpy as np
import tensorflow as tf
from tensorflow.keras import layers, models, optimizers
import gym

# --- Environment Setup ---
env = gym.make("CartPole-v1")
num_actions = env.action_space.n
num_states = env.observation_space.shape[0]

# --- Build Q-Network ---
def build_q_network(num_states, num_actions):
    model = models.Sequential([
        layers.Input(shape=(num_states,)),
        layers.Dense(64, activation='relu'),
        layers.Dense(64, activation='relu'),
        layers.Dense(num_actions, activation='linear')
    ])
    return model

q_network = build_q_network(num_states, num_actions)
optimizer = optimizers.Adam(learning_rate=0.001)
gamma = 0.99  # Discount factor

# --- Epsilon-greedy policy ---
def epsilon_greedy(state, epsilon=0.1):
    if np.random.rand() < epsilon:
        return np.random.randint(num_actions)
    q_values = q_network(np.array([state]), training=False)
    return np.argmax(q_values.numpy()[0])

# --- Training Loop ---
num_episodes = 500
for episode in range(num_episodes):
    state, _ = env.reset()
    total_reward = 0
    done = False

    while not done:
        action = epsilon_greedy(state)
        next_state, reward, terminated, truncated, _ = env.step(action)
        done = terminated or truncated
        total_reward += reward

        # Compute target (Bellman optimality)
        next_q = np.max(q_network(np.array([next_state]), training=False))
        target = reward + gamma * next_q * (1 - int(done))

        with tf.GradientTape() as tape:
            q_values = q_network(np.array([state]), training=True)
            q_value = q_values[0, action]
            loss = tf.square(target - q_value)

        grads = tape.gradient(loss, q_network.trainable_variables)
        optimizer.apply_gradients(zip(grads, q_network.trainable_variables))
        state = next_state

    print(f"Episode {episode+1}: Total Reward = {total_reward}")

env.close()
```

### Explanation

| Step              | Description                                            |
| ----------------- | ------------------------------------------------------ |
| **Environment**   | `CartPole-v1` — a simple control task                  |
| **Q-network**     | Approximates $Q(s, a)$ with a neural network           |
| **Loss function** | $(r + \gamma \max_{a'} Q(s', a') - Q(s, a))^2$         |
| **Policy**        | $\epsilon$-greedy: explore with probability $\epsilon$ |
| **Update**        | Gradient descent to minimize Bellman error             |

## Summary

| $Q^*(s, a)$               | Optimal action-value function                                                 |
| Bellman Equation          | Defines recursive relationship between Q-values                               |
| Bellman Optimality        | Used in Q-Learning and Deep Q-Networks                                        |

---

## Key Takeaways

* The **Q-function** quantifies how good an action is in a state.
* **Bellman equations** define the recursive learning target.
* **Deep Q-Learning** uses neural networks to approximate $Q^*(s, a)$ efficiently.
* Works well even in **stochastic environments** and **continuous tasks** when combined with experience replay and target networks.
