# Introduction to Reinforcement Learning (RL)

**Reinforcement Learning (RL)** is a subfield of **Machine Learning** where an **agent** learns to make decisions by **interacting with an environment** to achieve a goal.

Unlike supervised learning (where correct answers are given), RL relies on **trial and error** — learning through **rewards and punishments**.

---

## Intuition

> Reinforcement Learning mimics how humans and animals learn from experience.

An **agent** observes the **state** of the environment, takes an **action**, and receives a **reward**. Over time, it learns which actions yield the highest cumulative reward.

For example:
A robot learns to walk by repeatedly trying, falling, and receiving negative rewards for falling — and positive rewards for moving forward.

---

## Core Components

| Component                   | Description                                                    |                                           |
| --------------------------- | -------------------------------------------------------------- | ----------------------------------------- |
| **Agent**                   | Learner or decision-maker                                      |                                           |
| **Environment**             | The world the agent interacts with                             |                                           |
| **State ($s$)**             | Representation of the environment at a given time              |                                           |
| **Action ($a$)**            | Choice made by the agent                                       |                                           |
| **Reward ($r$)**            | Feedback signal from the environment                           |                                           |
| **Policy ($\pi(a            | s)$)**                                                         | Strategy the agent uses to choose actions |
| **Value Function ($V(s)$)** | Expected cumulative reward from state $s$                      |                                           |
| **Q-Function ($Q(s,a)$)**   | Expected cumulative reward from taking action $a$ in state $s$ |                                           |

---

## Reinforcement Learning Cycle

At each time step $t$:

1. The agent observes the current state $s_t$
2. Chooses an action $a_t$ according to its policy $\pi(a_t|s_t)$
3. Receives a reward $r_t$ and moves to the next state $s_{t+1}$
4. Updates its policy or value estimates based on the reward

---

## Objective Function

The agent’s goal is to **maximize the expected return**:

$$
J(\pi) = \mathbb{E}*{\pi} \left[ \sum*{t=0}^{\infty} \gamma^t r_t \right]
$$

Where:

* $\pi$: Policy (how actions are chosen)
* $\gamma$: Discount factor $(0 < \gamma \le 1)$
* $r_t$: Reward at time $t$

---

## Types of Reinforcement Learning

### 1. **Model-Free RL**

The agent learns directly from experience without knowing the environment’s dynamics.

#### A. **Value-Based Methods**

The agent learns **how good** each action is.

* **Q-Learning:**
  Updates Q-values using:
  $$
  Q(s_t, a_t) = Q(s_t, a_t) + \alpha [r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t)]
  $$
* Learns the *optimal action-value function*.

#### B. **Policy-Based Methods**

The agent directly learns the policy $\pi(a|s)$.

* Example: **REINFORCE algorithm**
  $$
  \nabla_{\theta} J(\theta) = \mathbb{E}*{\pi} [\nabla*{\theta} \log \pi_{\theta}(a|s) ; G_t]
  $$
* Useful for continuous or stochastic action spaces.

#### C. **Actor-Critic Methods**

Combines both approaches:

* **Actor:** Updates the policy
* **Critic:** Estimates value function to guide the actor

---

### 2. **Model-Based RL**

The agent builds an internal model of the environment:
$$
P(s', r \mid s, a)
$$
Then plans ahead using this model — similar to how humans mentally simulate future outcomes before acting.

Examples:

* **Monte Carlo Tree Search (MCTS)**
* **Dyna-Q**

---

## When to Use Reinforcement Learning

✅ Use RL when:

* The environment is **dynamic and sequential**
* Feedback is **delayed** (not after every action)
* The goal is to **maximize long-term reward**, not immediate gains

❌ Avoid RL when:

* Data is expensive or dangerous to collect (e.g., real-world robots)
* The task can be solved with supervised learning
* There’s no clear reward signal

## Common RL Functions and Concepts

| Function                         | Formula                                                                                                   | Description                                                                                      |
| -------------------------------- | --------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **Value Function**               | $V^{\pi}(s) = \mathbb{E}_{\pi}[R_t \mid s_t = s]$                                                         | Expected return from state $s$.                                                                  |
| **Q-Function**                   | $Q^{\pi}(s,a) = \mathbb{E}_{\pi}[R_t \mid s_t = s, a_t = a]$                                              | Expected return from taking action $a$ in state $s$.                                             |
| **Bellman Expectation Equation** | $V^{\pi}(s) = \sum_a \pi(a\mid s) \sum_{s',r} P(s',r\mid s,a) [r + \gamma V^{\pi}(s')]$                   | Recursive definition of $V^{\pi}(s)$ under policy $\pi$.                                         |
| **Bellman Optimality Equation**  | $V^{*}(s) = \max_a \sum_{s',r} P(s',r\mid s,a) [r + \gamma V^{*}(s')]$                                    | Defines the optimal value function.                                                              |
| **Policy Gradient**              | $\nabla_{\theta} J(\theta) = \mathbb{E}*{\pi}[\nabla*{\theta} \log \pi_{\theta}(a\mid s) ; Q^{\pi}(s,a)]$ | Gradient used in policy optimization (actor-critic variants replace $Q^{\pi}$ with an estimate). |

* OpenAI Five (Dota 2)

### 🏭 Robotics

* Robotic arm control
* Industrial process optimization

### 💰 Finance

* Portfolio management
* Algorithmic trading

### 🏥 Healthcare

* Personalized treatment recommendation
* Drug discovery simulations

* Network routing, data center cooling, and resource allocation

---

## Advantages

✅ Learns complex sequential tasks <br>
✅ Can operate with minimal supervision <br>
✅ Adapts to changing environments <br>
✅ Achieves superhuman performance in some domains <br>

---

## Limitations

⚠️ High sample complexity (requires lots of interactions) <br>
⚠️ Hard to tune hyperparameters (α, γ, exploration rate) <br>
⚠️ Reward design is non-trivial <br>
⚠️ Difficult to guarantee safety in real-world tasks <br>

## Conclusion

Reinforcement Learning enables machines to **learn optimal behavior through experience**, making it powerful for problems involving **sequential decisions** and **long-term goals**.

It bridges the gap between **supervised learning** (learning from data) and **control theory** (making sequential decisions), paving the way for intelligent agents that learn to **act, plan, and adapt** autonomously.
